Long-Range LoRa Telemetry (DogTracker)

Edge-to-Cloud IoT Architecture & RF Communication

Tech Stack C++, REST API, NMEA Parsing
RF Communication LoRa (Long Range) up to 10km
Hardware Arduino Micro IoT, GPS, BMP180
Cloud Services Google Sheets API / Maps

Project Context & Client Constraints

A local farmer approached me with a practical problem: his dog frequently escaped the property, making retrieval difficult across vast rural terrain. The client provided two Arduino Micro IoT boards and requested a custom tracking solution.

Cellular (GSM) coverage in the area was unreliable and required a paid subscription. Therefore, I designed a subscription-free, Edge-to-Cloud architecture relying on Long-Range RF communication (LoRa) to transmit telemetry data directly from the animal to a base station at the farm.

Distributed System Architecture

The system was split into two distinct physical nodes, each with specific power and communication constraints:

Mobile Edge Node (Transmitter)

A compact, battery-powered (3.7V) device attached to the collar. Its primary logic loop handled:

  • GPS Parsing: Extracting real-time coordinate data via NMEA sentences.
  • Biological Status: Reading a BMP180 sensor (temperature/pressure)—a specific client request to monitor the animal's physical state.
  • Payload Packing: Serializing the data and transmitting it via the LoRa module.

WiFi Gateway Node (Base Station)

Mounted at the client's home, this node acted as the bridge between the RF network and the Internet.

  • RF Reception: Equipped with a 50cm, high-gain (15dB) SMA antenna mounted on the roof, achieving a tested range of up to 10km.
  • Network Translation: The microcontroller received the LoRa payload, connected to the local WiFi network, and established a secure TCP/IP connection.

Serverless Cloud Dashboard (Google API)

To provide the client with a user-friendly, zero-cost interface accessible from their smartphone, I integrated the Gateway Node with the Google Sheets REST API.

Every 10 seconds, the Gateway dispatched an HTTP POST request containing the parsed GPS coordinates and sensor data. The Google Sheet acted as a live database, dynamically updating a custom Google Maps link. This allowed the client to open an app on their phone and instantly see a real-time breadcrumb trail of the animal's movement across the terrain.
